● 왜 필요한가

Claude Code 는 강력합니다. 혼자서 쓰면, 거기까지입니다.

한 명의 에이전트가 기획·코딩·테스트·리뷰를 다 떠안으면 결과는 뻔합니다. 속도는 느려지고, 버그는 쌓이고, 토큰은 재작업에 낭비됩니다. Claude 의 잠재력 80% 를 책상 위에 그대로 두는 셈이죠.

BEFORE 혼자 쓰는 Claude Code
  • ✕ 순차적 작업 — 생각하고, 만들고, 테스트하고, 리뷰하는 동안 그저 기다림
  • ✕ 사소한 수정에도 Opus 가 동원돼 토큰 낭비가 누적
  • ✕ 어려운 버그 앞에서 포기하거나, 같은 루프에 갇힘 — 검증 게이트 없음
  • ✕ 어떤 명령어가 어떤 에이전트와 어울리는지 외워야 함
WITH OMC 오케스트레이션된 팀
  • ✓ 최대 5명이 병렬로 — architect 가 설계하는 사이 executor 가 구현
  • ✓ 스마트 모델 라우팅 — 단순 조회는 Haiku, 추론은 Opus
  • ✓ Ralph 모드는 architect 검증 전까지 절대 멈추지 않음
  • ✓ 그냥 자연어로 말하면 OMC 가 알아서 모드를 골라줍니다

● 자주 묻는 질문

짧은 답변, 긴 안심.

설치 외에 추가 설정이 정말 필요 없나요?
네. /omc-setup 한 번이면 에이전트·스킬·훅이 모두 등록됩니다. 자연어로 일을 설명하면 OMC 가 적합한 모드를 골라요. autopilot · ralph 같은 키워드는 파워 유저용 단축어일 뿐, 외울 의무는 없습니다.
Max / Pro 구독이 필요한가요? Anthropic API 키만 있어도 되나요?
Claude Code CLI 가 깔려 있다면 둘 중 하나만 있으면 됩니다. Claude Max / Pro 구독이거나 Anthropic API 키. OMC 는 그 위에 얹히는 오케스트레이션 레이어라 별도 결제 없이 그대로 동작합니다.
토큰 절감은 어떤 원리로 이루어지나요?
스마트 모델 라우팅 덕입니다. 단순 검색·문서 작성 같은 일에는 Haiku, 복잡한 추론에는 Opus 를 자동 배정합니다. 결과적으로 같은 결과를 30–50% 적은 토큰으로 — 품질을 떨어뜨리지 않으면서요.
실무·프로덕션 코드에 써도 안전한가요?
네. security-reviewer 에이전트가 OWASP Top 10 · 시크릿 검사 · 안전하지 않은 패턴을 점검하고, architect 검증과 “완료 전 검증” 프로토콜이 게이트를 만듭니다. 그래도 모든 자동 생성 코드는 사람의 PR 리뷰를 거치는 걸 권장합니다 — OMC 가 그걸 더 쉽게 만들어 주는 도구입니다.
한국어로 프롬프트를 줘도 잘 되나요?
그럼요. /autopilot "로그인이 있는 투두 앱 만들어줘" 처럼 그냥 한국어로 말해도 동일하게 동작합니다. 명령어·키워드(autopilot, ralph, ulw)만 영문이고, 인자는 어떤 언어로 줘도 됩니다.
Autopilot 과 Ultrawork 의 차이는?
Autopilot 은 단계 게이트가 있는 순차 자율 실행입니다. Ultrawork 는 거기에 최대 병렬을 더한 것 — 프롬프트 앞에 ulw 만 붙이면 풀스택 기능이나 대규모 리팩토링이 3–5× 빠르게 끝납니다.
